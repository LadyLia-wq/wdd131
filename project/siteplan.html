Khalia Beauty Nails & Spa — Site Plan

A compact site plan for the website project. This single-page plan outlines the site name, purpose, audience scenarios, color and typography choices, wireframes, and testing checklist.

Site Name & Logo

Site Name: Khalia Beauty Nails & Spa

Short rationale: The name is the business owner's name and describes the service (nail care & beauty). It is simple, brandable, and local-market friendly.

Site Purpose

The website will act as a professional online presence for Khalia Beauty:

  • Introduce the salon and staff, show services and pricing
  • Display a gallery of nail designs and recent work
  • Allow customers to send inquiries via a contact/booking form
  • Highlight promotions and collect newsletter signups (localStorage)

User Scenarios

  1. Scenario A: A local customer wants to know which manicure options (regular, gel, acrylic) are available and their prices before calling. — They should find an easy-to-scan services page with prices and descriptions.
  2. Scenario B: A visitor wants to see recent work to decide whether the salon's style matches theirs. — They should view the gallery page with filters for "manicure", "pedicure", "bridal" and see large images with lazy loading.
  3. Scenario C: A returning customer wants to view store hours and contact details quickly on mobile. — The header/footer will include a persistent contact area and a mobile-first layout.

Contrast notes: primary is dark enough for headings and buttons. We'll run Lighthouse and DevTools color contrast checks and adjust if needed.

Typography

Fonts chosen (Google Fonts):

  • Poppins — Headings and brand wordmark (weights 600,700). Clean, geometric, modern look.
  • Open Sans — Body copy, form labels, and nav (weights 400,600). Good legibility on small screens.

Heading example — "Spa & Nail Services" (Poppins)

Body example — "We offer manicures, pedicures, and gel services with a focus on hygiene and custom designs." (Open Sans)

CSS approach: a small reset + utility classes. Keep stylesheet lean, remove unused rules. Use CSS variables for colors and spacing.

JavaScript Features (requirements mapping)

  • Multiple functions: initGallery(), loadPromotions(), handleFormSubmit()
  • DOM interaction: select elements, update gallery, show/hide modal, form validation
  • Conditional branching: toggle UI based on localStorage (e.g., "seenPromo")
  • Objects & arrays: gallery items stored as objects; use map, filter
  • Template literals: used exclusively when building HTML strings for injection
  • localStorage: save newsletter subscription, remember user's preferred gallery filter

Testing & Validation Checklist

  • HTML validation: all pages pass a standards-based validator (no structural errors)
  • CSS validation: no duplicate/unused rules; use DevTools CSS Overview
  • Accessibility: check color contrast, use semantic HTML (landmark elements), form labels
  • Performance: images optimized, lazy loading implemented, Lighthouse score > 80 mobile
  • JavaScript console: no runtime errors; functions tested in DevTools
  • SEO basics: meaningful title/meta descriptions, clean URLs, alt attributes for images
